# ANTIGRAVITY.md - Project Constitution

## 1. プロジェクト概要
- **プロジェクト名**: Baseball Oracle (ベースボール・オラクル)
- **目的**: 不特定多数のユーザーが参加し、野球の試合展開（5回裏終了時 & 試合終了時）を予想して競い合うWebアプリケーション。
- **コア体験**: "直感的な予想" と "美しいUI" による熱狂的な観戦体験の拡張。

## 2. ターゲット & 利用シーン
- **ターゲット**: 野球ファン、データ好き、または単に盛り上がりたい不特定多数のユーザー。
- **利用シーン**: 試合開始前のワクワクする時間にスマホで予想を投稿。試合中はリアルタイムで結果を見守る。

## 3. MVP要件 (Minimum Viable Product)
### 機能
1.  **試合一覧表示**: 本日開催の試合を表示。
2.  **予想投稿**:
    -   **締め切り**: 試合開始時刻まで。
    -   **予想内容**: 「5回裏終了時点のスコア」と「最終スコア」の2点予想。
3.  **簡易認証**: ユーザー名のみのゲストログイン（MVP段階）、またはメール認証。
4.  **ランキング**: 予想的中率またはポイントによるランキング表示。

### デザイン & Vibe
- **テーマ**: "Stadium Night Lights"
    -   ダークモード基調。
    -   フィールドの緑、ネオンのアクセント、グラスモーフィズム（すりガラス効果）を多用。
    -   数字は大きく、読みやすく、ドラマチックに。
- **レスポンシブ**: モバイルファースト。

## 4. 技術スタック
- **Frontend**: Next.js (App Router), React, Tailwind CSS, Framer Motion (アニメーション)
- **Backend/DB**: Supabase (PostgreSQL, Auth, Realtime)
- **Hosting**: Vercel (想定)

## 5. 開発フェーズ
1.  **Phase 1 (Now)**: Next.js プロジェクト作成、デザインシステム構築、モックデータによるUI実装。
2.  **Phase 2**: Supabase連携、実際のデータ（または管理画面からの入力）に基づく動的表示。
3.  **Phase 3**: 公開に向けたセキュリティ設定とデプロイ。

## 6. 特記事項
- **ユーザー体験優先**: 登録や入力の「面倒くささ」を極限まで排除する。
- **拡張性**: 将来的には「誰がホームランを打つか」などの予想項目追加を見越した設計にする。
